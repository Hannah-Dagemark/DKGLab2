The first suggestion concerns the UX design of the filtering functionality when thinking of controls, feedback and pliancy. Currently, there is no way to reset all the filters at once, requiring users to manually deselect each option. This limits the pliancy of the interface, as it reduces both flexibility and ease of recovery. In particular, the absence of a reset function weakens dynamic hinting, since users are not provided with clear interactive cues for how to quickly modify or undo their selections. Introducing a “reset filters” feature would improve efficiency, strengthen user control, and support smoother interaction. 

Additionally, such a feature would improve feedback, especially visual and dynamic feedback. Some filters may not clearly communicate whether they are active, which increases the risk of misinterpretation. Since users may overlook or misunderstand the system state, providing a reset option acts as a form of feedback redundancy and a reliable fallback mechanism.

Another improvement relates to the site’s logo in terms of controls, layout, and pliancy. In most web interfaces, the logo functions as an “escape hatch”, allowing users to quickly return to the homepage.  This behavior is supported by static object hinting, as users recognise the logo as an interactive element based on established design conversions. This also provides implicit visual feedback, as the design itself signals functionality without additional instruction. Making the logo clickable would therefore align with user expectations and improve navigation. 

There is also an issue with the feedback provided by the time and cost sliders. When these sliders are set to their minimum values, they display “0,” which can be interpreted as filtering for the lowest cost or shortest time. However, this is not the case,  instead the filter is disabled entirely. This creates misleading visual feedback and incorrect static hinting, as the displayed value suggests a system state that does not match the actual behaviour. Furthermore, the lack of clear dynamic feedback when adjusting the sliders makes it difficult for users to understand how their input affects the results.
This issue could be resolved in two ways. One option is to display a neutral value such as “–” to indicate that no filter is applied, improving both feedback clarity and consistency of static hinting. Another option is to redefine “0” as the lowest possible value, making the control behaviour align with user expectations. However, this would require an alternative method for disabling the filter, in order to preserve pliancy and maintain user control over the system.
