The first suggestion we have is regarding the UX design of the filtering functionality. There is currently no way of 'resetting' filters, requiring the user to manually deselect each filter. Adding a filter reset would also provide feedback redundancy, as some filters may not always clearly communicate their filtering status. We will elaborate on an example of this later, but in general one can not expect to always catch possible feedback missinterpretation, and having redundancy or backup functionality provides a safety net in such cases.

The logo image also typically acts as an escape hatch, in and of itself the logo of a website typically has static hinting that it is interactable, in large due to conditioning as this has become the web standard. Making the logo an escape hatch would make the site easier to navigate, whilst also complying with the inherent static hinting that comes with being the site's main logo.

There is also a problem with the slider feedback. Setting the sliders for time and cost to 'nothing' shows a price of 0 and a time of 0. This feedback can be interpreted as filtering for 'lowest cost' and 'quickest time'. However, these do in fact not filter for either of those, instead it removes the respective filter alltogether. This is an example of feedback missinterpretation, which we discussed earlier. This can be resolved in two ways, either by making the default setting display '-' instead of '0', or by making the '0' filter filter for lowest possible. Doing the latter, however, would require a new alternate way for users to disable said filters.
