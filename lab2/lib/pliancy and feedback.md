When a user clicks on a list item, the light darkens showing signs on visual feedback as well as dynamic visual hinting. 


The filter function on the website visually communicates clearly that it is filtering items, but doesn't inform the user of the details of the changes. When multiple filters are layered on top of eachother, it is impossible to tell what filter might be weighing into the placement of a certain item. 

A way to improve filter feedback would be to make the effect of the filter visually communicated. An example would be using visual feedback which tells the user how accurate the different list items are to the filter, or separating the filtered items into categories with different ranges of accuracy. 

When it comes to pliancy, the selection screen also does a good job at communicating it's functionality through visual hinting and communicative responses to user actions. This is exemplified in the Price and Time sliders.
